## Sources for Theme 2: What is the history and evolution of MCP?

- **Source 1:** [The Surprising Origins of the Model Context Protocol | by Kyle Stratis](https://kylestratis.medium.com/the-surprising-origins-of-the-model-context-protocol-868d640ac7c6)
  - Snippet: "MCP was born out of an internal project at Anthropic. David Soria Parra, a software engineer at Anthropic, was using Claude Desktop to assist him with his day-to-day work: writing developer tools. But he was frustrated by having to constantly copy and paste code between Claude Desktop and his code editor." (Aug 13, 2025)
  - Snippet: "He had also been working on a Language Server Protocol (LSP) project, which inspired him to take a similar approach to this problem."
  - Snippet: "David noticed that writing integrations for LLMs, especially for more than one, created the MxN problem... With the LSP project still in the back of his mind, it dawned on David that an LSP-like protocol could be the perfect solution to the MxN problem."
  - Snippet: "David brought his idea to Justin Spahr-Summers, and the two began setting the foundations for what would become MCP. Later that year, before MCP’s initial release, Anthropic had an internal hackathon. Several teams adopted MCP right away..."

- **Source 2:** [Model Context Protocol - Wikipedia](https://en.wikipedia.org/wiki/Model_Context_Protocol)
  - Snippet: "Introduced November 25, 2024; 12 months ago (2024-11-25)"
  - Snippet: "The Model Context Protocol (MCP) is an open standard, open-source framework introduced by Anthropic in November 2024..."
  - Snippet: "Following its announcement, the protocol was adopted by major AI providers, including OpenAI and Google DeepMind."
  - Snippet: "MCP was announced by Anthropic in November 2024 as an open standard for connecting AI assistants to data systems such as content repositories, business management tools, and development environments."
  - Snippet: "The protocol was created at Anthropic by developers David Soria Parra and Justin Spahr-Summers."
  - Snippet: "MCP aims to address the challenge of information silos and legacy systems. Before MCP, developers often had to build custom connectors for each data source or tool, resulting in what Anthropic described as an 'N×M' data integration problem."
  - Snippet: "Earlier stop-gap approaches—such as OpenAI's 2023 'function-calling' API and the ChatGPT plug-in framework—solved similar problems but required vendor-specific connectors."
  - Snippet: "In March 2025, OpenAI officially adopted the MCP, following a decision to integrate the standard across its products..."
  - Snippet: "Demis Hassabis, CEO of Google DeepMind, confirmed in April 2025 MCP support in the upcoming Gemini models and related infrastructure."

- **Source 3:** [Engineering AI systems with Model Context Protocol · Raygun Blog](https://raygun.com/blog/announcing-mcp/)
  - Snippet: "On November 26, 2024, Anthropic released the Model Context Protocol (MCP)—an open standard for data exchange between applications and data sources."

- **Source 4:** [A Deep Dive Into MCP and the Future of AI Tooling](https://a16z.com/a-deep-dive-into-mcp-and-the-future-of-ai-tooling/)
  - Snippet: "Model Context Protocol (MCP), introduced in November 2024, has gained significant traction within developer and AI communities as a potential solution."
  - Snippet: "The idea is not new; MCP took inspiration from the LSP (Language Server Protocol)."
  - Snippet: "Where MCP extends beyond LSP is in its agent-centric execution model: LSP is mostly reactive (responding to requests from an IDE based on user input), whereas MCP is designed to support autonomous AI workflows."

- **Source 5:** [Introducing the Model Context Protocol - Anthropic](https://www.anthropic.com/news/model-context-protocol)
  - Snippet: "Today, we're open-sourcing the Model Context Protocol (MCP), a new standard for connecting AI assistants to the systems where data lives, including content repositories, business tools, and development environments. Its aim is to help frontier models produce better, more relevant responses. Nov 25, 2024"
  - Snippet: "Early adopters like Block and Apollo have integrated MCP into their systems, while development tools companies including Zed, Replit, Codeium, and Sourcegraph are working with MCP to enhance their platforms..."

- **Source 6:** [One Year of MCP: November 2025 Spec Release](http://blog.modelcontextprotocol.io/posts/2025-11-25-first-mcp-anniversary/)
  - Snippet: "Today, MCP turns one year old... we’re also releasing a brand-new MCP specification version." (November 25, 2025)
  - Snippet: "Within the same timeframe, we saw the number of active MCP servers go from just a few experimental ones to thousands."
  - Snippet: "That’s why we also launched the MCP Registry earlier this year. It’s the central index for all available MCP servers that now has close to two thousand entries since its announcement in September."
  - Snippet: "To keep this pace sustainable, we spent some time thinking through and putting together a governance structure... Potential contributors now also know how to jump in through formal Working and Interest Groups."
  - Snippet: "The latest release of the MCP specification ships with a number of highly-anticipated features... Support for Task-based Workflows... Simplified Authorization Flows... Security and Enterprise Features... Extensions... URL Mode Elicitation... Sampling with Tools: Agentic Servers... Developer Experience Improvements."

- **Source 7:** [Roadmap - Model Context Protocol](https://modelcontextprotocol.io/development/roadmap)
  - Snippet: "Last updated: 2025-10-31. This page outlines our priorities for the next release on November 25th, 2025..."
  - Snippet: "Currently, MCP is built around mostly synchronous operations. We’re adding async support to allow servers to kick off long-running tasks..."
  - Snippet: "As organizations deploy MCP servers at enterprise scale, we’re addressing challenges around horizontal scaling."
  - Snippet: "We’re enabling servers to advertise themselves through `.well-known` URLs..."
  - Snippet: "As MCP has grown, valuable patterns have emerged for specific industries and use cases. Rather than leaving everyone to reinvent the wheel, we’re officially recognizing and documenting the most popular protocol extensions."
  - Snippet: "We’re introducing a clear tiering system for SDKs based on factors like specification compliance speed, maintenance responsiveness, and feature completeness."
  - Snippet: "The MCP Registry launched in preview in September 2025 and is progressing toward general availability."
